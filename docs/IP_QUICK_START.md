# IP_QUICK_START — Навигация по артефактам IP (x0tta6bl4)

## 1. Быстрый обзор
Портфель интеллектуальной собственности проекта разделён на подтверждённые, неподтверждённые и кандидаты.

## 2. Ключевые файлы
| Файл | Назначение |
|------|------------|
| PATENT_PORTFOLIO.md | Полная инвентаризация объектов IP |
| FTO_MATRIX.md | Freedom-To-Operate анализ по компонентам |
| IP_POLICY.md | Политика лицензирования, вкладов и разглашения |
| IP_FINAL_REPORT.md | Итоговый статус, риски, приоритеты |
| BENCHMARK_README.md | Методика подтверждения производственных заявок |
| QUANTUM_INTEGRATION_SUMMARY.md | Состояние квантовых и PQC интеграций |

## 3. Категории объектов
- CONFIRMED: Существующие проверенные заявки / реализации
- UNCONFIRMED: Требуют юридической проверки или бенчмарков
- CANDIDATE: Идеи для подачи (внутренние подтверждения)

## 4. Первые действия (если вы новый участник)
1. Прочитать PATENT_PORTFOLIO.md
2. Сверить риски в IP_FINAL_REPORT.md
3. Проверить есть ли активные блокеры (φ-QAOA, RSA claim)
4. Ознакомиться с метриками в BENCHMARK_README.md
5. Соблюдать правила раскрытия из IP_POLICY.md

## 5. Метрики для подтверждения
| Claim | Метрика | Способ проверки |
|-------|---------|-----------------|
| NMP improvement | % latency reduction | Routing бенчмарк (baseline vs enhanced) |
| φ-QAOA uplift | approximation ratio / runtime | MaxCut / Portfolio selection задачи |
| 7653× performance | runtime vs classical baseline | Симуляция + физический backend |
